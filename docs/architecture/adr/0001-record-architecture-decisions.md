# ADR 0001: Record Architecture Decisions

Status: Accepted

## Context

The project accumulated important choices about naming, profiles, safety, refresh behavior, and tree grouping during iterative development.

## Decision

Use ADRs for future consequential decisions. Keep current facts in the owning product, architecture, engineering, operations, or AI gotcha documents instead of a catch-all facts file.

## Consequences

- New irreversible or cross-cutting choices should get an ADR.
- Routine implementation details should stay close to code or tests.
